LockerJob()

The Enter() and Exit() pattern as mentioned above during this even other thread won't be able to access the variable {x} until 1 seconds is passed(because of thread sleep)

 _padlock.EnterReadLock();
 Console.WriteLine($"Entered read lock, x= {x}");
 Thread.Sleep(1000);

Here(in mddle of Enter and Exit section) in case we now want to get Write Lock, we can't do that. that will throw exception because this operation by design is prone to deadlocks.

_padlock.ExitReadLock();
Console.WriteLine($"Exited read lock, x= {x}");
